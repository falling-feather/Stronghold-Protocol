## 🎯 功能开发计划

### 阶段一：盟约叠层机制

#### 1.1 盟约系统设计

**参考PRTS Wiki的盟约机制：**

- **盟约类型**：每个玩家在游戏开始时选择一个盟约
- **叠层机制**：通过特定操作获得叠层，叠层提供增益效果
- **叠层上限**：每种盟约有最大叠层数
- **叠层消耗**：某些操作会消耗叠层

#### 1.2 实现方案

**数据结构设计：**

```typescript
// src/types/index.ts
export type CovenantType = 
  | '炎佑'      // 炎国机关术
  | '碎铳之簧'  // 铳械流派
  | '余音'      // 音律流派
  | '其他';     // 可扩展

export interface Covenant {
  id: string;
  name: string;
  type: CovenantType;
  maxStacks: number;
  currentStacks: number;
  effects: CovenantEffect[]; // 根据叠层数触发的效果
}

export interface CovenantEffect {
  stackThreshold: number; // 触发该效果的叠层数
  effectType: 'stat_boost' | 'special_ability' | 'resource_bonus';
  description: string;
  // 具体效果数据...
}
```

**核心功能：**

1. **叠层获取机制**

   - 击杀特定敌人获得叠层
   - 完成特定操作获得叠层
   - 每波次开始获得基础叠层
2. **叠层效果系统**

   - 根据叠层数提供属性加成
   - 解锁特殊能力
   - 资源获取加成
3. **UI显示**

   - 盟约选择界面（游戏开始前）
   - 叠层显示（顶部状态栏）
   - 效果说明面板

**实现文件：**

- `src/core/CovenantSystem.ts` - 盟约系统核心逻辑
- `src/config/covenants.ts` - 盟约配置数据
- `src/view/CovenantUI.ts` - 盟约UI组件

### 阶段二：角色选择系统

#### 2.1 角色选择机制

**参考PRTS盟约的角色选择：**

- **初始角色池**：游戏开始时从特定角色池中选择
- **定向甄选**：根据盟约类型提供推荐角色
- **角色限制**：某些角色只能在特定盟约下使用

#### 2.2 实现方案

**数据结构：**

```typescript
// src/types/index.ts
export interface OperatorSelection {
  operatorId: string;
  covenantRestriction?: CovenantType[]; // 限制可用的盟约
  isRecommended?: boolean; // 是否为推荐角色
}

export interface SelectionPool {
  poolId: string;
  name: string;
  operators: OperatorSelection[];
  maxSelect: number; // 最多选择数量
}
```

**核心功能：**

1. **选择界面**

   - 显示可选角色池
   - 显示盟约推荐角色
   - 选择确认机制
2. **限制系统**

   - 检查盟约限制
   - 显示不可选原因
3. **数据持久化**

   - 保存玩家选择
   - 在商店中优先出现已选角色

**实现文件：**

- `src/core/SelectionSystem.ts` - 角色选择系统
- `src/config/selectionPools.ts` - 角色池配置
- `src/view/SelectionUI.ts` - 选择界面

### 阶段三：联机玩法

#### 3.1 联机架构设计

**技术选型：**

- **WebSocket**：实时通信（推荐使用 Socket.io）
- **房间系统**：玩家创建/加入房间
- **同步机制**：状态同步、延迟补偿

#### 3.2 实现方案

**服务器端（需要新建）：**

```
server/
├── index.ts          # WebSocket服务器
├── RoomManager.ts    # 房间管理
├── GameSync.ts       # 游戏状态同步
└── PlayerManager.ts  # 玩家管理
```

**客户端：**

- `src/core/NetworkManager.ts` - 网络通信管理
- `src/core/MultiplayerEngine.ts` - 多人游戏引擎
- `src/view/MultiplayerUI.ts` - 联机UI（房间列表、等待界面等）

**核心功能：**

1. **房间系统**

   - 创建房间（设置密码、最大人数）
   - 加入房间（房间号/密码）
   - 房间列表浏览
   - 玩家准备状态
2. **游戏同步**

   - 波次同步（所有玩家同时开始战斗）
   - 敌人同步（共享敌人路径和状态）
   - 生命值共享（所有玩家共享生命值）
   - 资金独立（每个玩家独立管理）
3. **协作机制**

   - 共享地图视野
   - 可以查看队友部署
   - 聊天系统（可选）

**实现步骤：**

1. **Phase 1：基础联机**

   - 实现房间创建/加入
   - 实现基础同步（波次、生命值）
2. **Phase 2：完整同步**

   - 敌人状态同步
   - 角色部署同步（可选，或保持独立）
3. **Phase 3：优化**

   - 延迟补偿
   - 断线重连
   - 性能优化

### 阶段四：技能激活系统

#### 4.1 技能系统完善

**当前状态：**

- ✅ 技力积累系统已实现
- ❌ 技能激活未实现
- ❌ 技能效果未实现

#### 4.2 实现方案

**技能类型：**

1. **主动技能**：手动激活（点击技能按钮）
2. **自动技能**：达到技力后自动激活
3. **持续技能**：激活后持续一段时间

**核心功能：**

1. **技能激活**

   - 检查技力是否足够
   - 执行技能效果
   - 消耗技力
2. **技能效果实现**

   - 属性加成（攻击力、防御力等）
   - 范围攻击
   - 特殊效果（治疗、控制等）
3. **UI交互**

   - 技能按钮（主动技能）
   - 技能冷却显示
   - 技能激活动画

**实现文件：**

- `src/core/SkillSystem.ts` - 技能系统
- `src/core/SkillEffects.ts` - 技能效果实现
